# Engineering Report — Iteration 100

## Change Summary

Added fractional differentiation features (AFML Ch. 5) to the feature pipeline. Implemented fixed-window fracdiff algorithm without external dependency (fracdiff package incompatible with numpy 2.0).

## Implementation Details

**New module**: `src/crypto_trade/features/fracdiff.py`
- `_fracdiff_weights(d, window)`: computes weights w_k = -w_{k-1} * (d-k+1)/k
- `fracdiff_series(series, d, window)`: applies fixed-window fracdiff via dot product
- `add_fracdiff_features(df)`: adds 2 columns to DataFrame
  - `stat_fracdiff_close_04`: fracdiff(log(close), d=0.4, window=100)
  - `stat_fracdiff_volume_04`: fracdiff(log(volume), d=0.4, window=100)

**Registration**: Added to `features/__init__.py` as group "fracdiff"

**Parquet regeneration**: Regenerated BTC and ETH parquets with all 9 groups → 195 features (was 185 in baseline). After removing fracdiff, regenerated with 8 groups → 193 features. The 8 extra non-fracdiff features come from feature code changes since the baseline parquets were originally generated.

## IMPORTANT: Parquet State Change

The baseline parquets (185 features) have been REPLACED by 193-feature parquets. The 8 extra features were always in the code but weren't in the original parquets because they were generated by an older code version. Future iterations will discover 193 features instead of 185.
